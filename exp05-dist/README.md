Overview
===

In this experiment I am aiming to see what classification accuracy is like by comparing the distribution of kmer frequencies for a particular instance with the mean distribution of kmer frequencies for each genera.

For example, we can take all instances corresponding to the class "Theretra" and take the arithmetic mean of all attributes, effectively creating a "mean instance". We can construct a density plot from this instance (a "mean distribution"), which is shown below in black:

![alt](https://raw.githubusercontent.com/chrispy645/barcoding/master/exp05-dist/figure/dist.png?token=2417792__eyJzY29wZSI6IlJhd0Jsb2I6Y2hyaXNweTY0NS9iYXJjb2RpbmcvbWFzdGVyL2V4cDA1LWRpc3QvZmlndXJlL2Rpc3QucG5nIiwiZXhwaXJlcyI6MTQwODE3NDA0Mn0%3D--842f7cc18518f63ea3e4580f90e5257897f1237a)

(Imagine each attribute lies somewhere on the x-axis and its frequency on the y-axis.)

A random instance in the dataset is also shown in the plot as a dotted red line.

So one way to classify what class a random instance should belong to, is to compute the Euclidean distance between all of its attributes and all of the attributes of some "mean distribution". If we have 300 different class values (genera), then we will be computing the Euclidean distance between the random instance and 300 different "mean distributions" and choosing the class that minimizes this distance.

So long as we have a training set with mean instances (i.e. every instance in the training set is the mean instance of its class) then we can simply use k-nearest-neighbours (where k = 1) to find the class whose mean distribution closely matches that of the random instance.
